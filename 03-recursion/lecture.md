# Recusion คืออะไรหว่า

ถ้าเราไปแปลตามพจนานุกรมแล้วมันจะแปลได้ว่า การเรียกตัวเองซ้ำ หรือ ฟังชั่นที่เรียกตัวมันเอง ไม่ว่าจะแปลว่าอะไรก็แล้วตามแต่หัวใจหลักของมันคือการ simplify ปัญหาให้เป็นเวอร์ชั่นที่ simplest ที่สุดและไอเวอร์ชั่นที่มัน simplest ที่สุดเราเรียกมันว่าเป็น base case ก็คือเราจะ simplify มันจนกว่าจะถึง case นี้นะเพราะถ้าไม่มีละก็เราก็ไม่รู้ว่าเราจะต้อง simplify มันลงไปถึงตรงไหนและจะทำให้เกิด infinity simplify/call ขึ้นมา จากคำอธิบายข้างต้นมันก็เกิดคำถามต่อมาว่าแล้วเราจะ simplify มันอย่างไรล่ะ เราเรียกสิ่งๆนี้ว่า recursive case ลองมาดูตัวอย่างกันดีกว่า

ให้หาผลรวมตั้ง 1 ถึง n และสมมุติว่า n = 5 ก็จะได้ว่า 0 + 1 + 2 + 3 + 4 + 5 = 15 <br>
เรามาเขียนฟังชั่นที่แก้โจทย์ๆนี้กันดีกว่า

Goal ของเราคือการรับค่า n และหาผลรวมตั้งแต่ 1 ถึง n หลังจากนั้นให้คืนค่าผลลัพธ์ออกมา

```py
def sum(n: int) -> int
```

เรารู้ว่า sum(5) = 5 + 4 + 3 + 2 + 1 + 0 สังเกตุไหมว่าเลขมันลดลงแบบมี pattern ของมันซึ่งก็คือมันจะลดลงไปทีละหนึ่งไปเรื่อยๆจนกว่าจะถึง 0 เราก็จะได้ว่า

```txt
15 เกิดจากการเอา 5 + 10
10 เกิดจาก 4 + 6
6 เกิดจาก 3 + 3
3 เกิดจาก 2 + 1
1 เกิดจาก 1 + 0
```

กล่าวคือ x = x + sum(x-1)

```txt
recursive case: n - 1
base case: n == 0
```

โค้ดเต็มๆ

```python
def sum(n: int) -> int:
   if n == 0:
    return 0
  return n + sum(n - 1)
```
